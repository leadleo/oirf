# External framework standard

Client-facing report frameworks should be concise and usable directly.

## Fixed Word format

The Word version is required. Use this fixed structure:

1. Cover page
   - centered project/report title
   - optional client/project entity
   - optional institution/author
   - date/version if available

2. Contents page
   - title: `目录`
   - first-level sections in `第一节/第一章` format
   - second-level items in `一、二、三` format

3. Report framework detail
   - title: `报告框架`
   - repeat each first-level section
   - write concise subsection descriptions in one sentence each
   - avoid internal methodology language

Use a restrained business document style: black text, clear hierarchy, no decorative over-design.

## Style

- Use a report title and contents-style framework.
- Prefer “第一节/第一章 + 一、二、三” for Chinese consulting/government reports.
- Include concise descriptions under each subsection when needed.
- Avoid long methodology explanations.
- Avoid internal implementation language.

## Do not expose

- L1-L5
- D1-D8
- JSON keys
- confidence scores
- internal routing
- hidden assumptions
- pricing, budget, quote, or amount information

## Feasibility-style example pattern

Title:

`深圳高性能医疗器械国家研究院有限公司可行性研究报告框架`

Contents:

1. 第一节 总论
   - 项目背景
   - 研究目标与范围
   - 投资亮点
   - 主要风险

2. 第二节 标的公司所属行业分析
   - 医疗器械行业
   - 医疗器械研发服务行业
   - 医疗器械CDMO与转化服务行业
   - 医疗影像行业
   - 体外诊断行业
   - 生物医用材料行业

3. 第三节 标的公司经营情况
   - 基本情况及历史沿革
   - 人员与组织情况
   - 主营业务与服务能力
   - 科研生产与创新能力
   - 商业模式与客户结构
   - 发展战略与技术规划
   - 知识产权与专利
   - 国家级承揽项目总览

4. 第四节 历史经营收益情况
   - 整体收入概览
   - 年度营收分析
   - 年度费用分析
   - 资产负债分析
   - 应收账款分析

5. 第五节 收益预测及合理性分析
   - 营业收入预测
   - 成本费用预测
   - 估值分析
   - IRR收益测算
   - 投资可行性总结

## Government research pattern

1. 调研背景与基本情况
2. 发展现状与主要成效
3. 存在问题
4. 原因分析
5. 对策建议
6. 城市亮点与标杆经验
