# Framework generation rules

## Formula

Report framework =

research object + industry context + research intent + depth + caliber constraints + engineering dimensions.

## Intent routing

### 行业全景/市场洞察

External structure:

1. 行业定义与发展背景
2. 政策与宏观环境
3. 产业链与价值链
4. 市场规模与竞争格局
5. 发展趋势与风险

### 细分市场/产品机会

External structure:

1. 目标市场边界
2. 市场空间与增长逻辑
3. 用户场景与需求痛点
4. 产品、价格与渠道
5. 竞争格局与机会判断

### 战略规划/市场进入/区域产业定位

External structure:

1. 项目背景与研究目标
2. 外部环境与产业机会
3. 区域/主体资源禀赋
4. 重点赛道筛选
5. 标杆案例与模式借鉴
6. 产业定位与实施路径

### 商业尽调/投资判断

External structure:

1. 投资标的及行业背景
2. 市场空间与增长驱动
3. 竞争格局与商业模式
4. 经营质量与财务表现
5. 风险因素与验证事项
6. 投资判断或待尽调事项

### 可行性研究/项目论证/经济影响评估

External structure:

1. 总论
2. 所属行业分析
3. 项目/标的经营或建设条件
4. 历史经营或基础条件分析
5. 收益预测、经济影响或合理性分析
6. 可行性总结

### 竞争格局/排名背书/评价体系

External structure:

1. 研究范围与口径
2. 候选企业/品牌/对象筛选
3. 评价指标与数据来源
4. 竞争格局或排名测算
5. 市场地位验证
6. 敏感性测试与附录说明

### 政府调研/政策建议

External structure:

1. 调研背景与基本情况
2. 发展现状与主要成效
3. 存在问题
4. 原因分析
5. 对策建议
6. 城市亮点或标杆经验

## Depth

- 简易: 3-5 sections, minimal module set.
- 常规: 6-8 sections, formal consulting/report structure.
- 深度: 8-12 sections or feasibility/audit-style structure, with evidence/model/verification placeholders.

## Compound wording

Do not produce separate frameworks in v1. Integrate secondary intent as a chapter or module.
